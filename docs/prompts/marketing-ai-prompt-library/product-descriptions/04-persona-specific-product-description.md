# Persona-specific product description

**Prompt Name:** Persona-specific product description  
**Category:** Product descriptions  
**Persona/Role:** Product marketing segmentation lead  
**Goal:** Adapt a product description for a specific buyer or user persona without changing approved claims.  
**Prompt Text:**

```text
Context: [PRODUCT NAME] serves [AUDIENCE]. We need copy tailored to [PERSONA], whose main pain is [PERSONA PAIN]. Approved product facts are [APPROVED PRODUCT FACTS].
Objective: Write a persona-specific product description that keeps the same product truth but changes emphasis for this persona.
Style: Clear, concrete and relevant to the persona's workflow.
Tone: [TONE].
Audience: [PERSONA].
Response: Return Persona Insight, Headline, Short Description, 3 Persona Benefits, Objection Response and CTA. Do not invent capabilities, integrations or persona research.
```

**Variables/Placeholders:** `[PRODUCT NAME]`, `[AUDIENCE]`, `[PERSONA]`, `[PERSONA PAIN]`, `[APPROVED PRODUCT FACTS]`, `[TONE]`  
**Example Output:**

**Persona Insight:** Finance leaders need reporting confidence before leadership asks for answers.  
**Headline:** Give finance a clearer revenue view before close  
**Short Description:** Atlas Analytics helps finance teams organize recurring revenue movement, owner follow-up and close-week reporting in one shared workflow.  
**Persona Benefits:** See account-level movement earlier; reduce spreadsheet handoffs; align reporting owners before review meetings.  
**Objection Response:** Built to support finance workflows, not replace financial judgment.  
**CTA:** See finance workflows

**Best Practices/Tips:** Start from approved product facts, then change emphasis for the persona. Keep the objection response factual and non-defensive. Validate persona pain with current research or sales notes before publishing.
