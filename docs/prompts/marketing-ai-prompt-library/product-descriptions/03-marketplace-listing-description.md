# Marketplace listing description

**Prompt Name:** Marketplace listing description  
**Category:** Product descriptions  
**Persona/Role:** Partner marketing copywriter  
**Goal:** Create a concise listing for a marketplace or integration directory.  
**Prompt Text:**

```text
Context: [PRODUCT NAME] appears in [MARKETPLACE OR DIRECTORY]. It is for [AUDIENCE] who need [USE CASE]. Approved capabilities are [APPROVED CAPABILITIES].
Objective: Write a marketplace listing description.
Style: Direct, scannable and compliance-safe.
Tone: [TONE].
Audience: [AUDIENCE].
Response: Return Short Description under [SHORT LIMIT] characters, Long Description under [LONG LIMIT] words, Key Benefits and Setup Note. Do not invent partner certifications or integration behavior.
```

**Variables/Placeholders:** `[PRODUCT NAME]`, `[MARKETPLACE OR DIRECTORY]`, `[AUDIENCE]`, `[USE CASE]`, `[APPROVED CAPABILITIES]`, `[TONE]`, `[SHORT LIMIT]`, `[LONG LIMIT]`  
**Example Output:**

**Short Description:** Connect recurring revenue data to clearer finance reporting workflows.  
**Long Description:** Atlas Analytics helps SaaS finance teams review subscription movement, account ownership and reporting tasks in one workflow built for close preparation.  
**Key Benefits:** Shared revenue views; owner-level reporting tasks; cleaner renewal planning.  
**Setup Note:** Connect your approved billing and CRM data sources before building reports.

**Best Practices/Tips:** Check each marketplace's character limits. Verify integration names and setup steps. Keep the long description readable on small directory cards.
