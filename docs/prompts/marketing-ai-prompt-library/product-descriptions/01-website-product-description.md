# Website product description

**Prompt Name:** Website product description  
**Category:** Product descriptions  
**Persona/Role:** Product marketing manager  
**Goal:** Write clear website copy for a product or feature page.  
**Prompt Text:**

```text
Context: [PRODUCT NAME] helps [AUDIENCE] solve [CUSTOMER PROBLEM] by [CORE CAPABILITY]. Approved differentiators are [DIFFERENTIATORS].
Objective: Write a product description for a website section.
Style: Specific, polished and conversion-aware.
Tone: [TONE].
Audience: [AUDIENCE].
Response: Return Headline, Subheadline, 3 Benefit Bullets, Proof Line and CTA. Do not invent integrations, outcomes or technical capabilities.
```

**Variables/Placeholders:** `[PRODUCT NAME]`, `[AUDIENCE]`, `[CUSTOMER PROBLEM]`, `[CORE CAPABILITY]`, `[DIFFERENTIATORS]`, `[TONE]`  
**Example Output:**

**Headline:** Revenue reporting that stays ready for close week  
**Subheadline:** Atlas Analytics helps SaaS finance teams turn subscription data into shared revenue views without rebuilding spreadsheets every month.  
**Benefit Bullets:** Spot account-level movement sooner; keep owners aligned on reporting tasks; give leadership a clearer view before close.  
**Proof Line:** Built for finance teams managing recurring revenue complexity.  
**CTA:** Explore Atlas Analytics

**Best Practices/Tips:** Keep claims tied to approved differentiators. Use the customer's language for the problem. Avoid listing features before the value is clear.
